# 첨부된 hwpx 파일을 xml 로 분류
# 페이지별로 세부적으로 분류 
# hwpx zip 파일 구조는 아래와 같아. 
  1. 최상위 구조 및 메타 정보
mimetype: 이 파일이 HWPX 패키지임을 명시하는 짧은 텍스트 파일입니다. (application/hwp+zip이 기록되어 있습니다.)

META-INF/ 폴더:

manifest.xml: 압축 파일 내부에 포함된 모든 파일의 경로와 각각의 미디어 타입(MIME Type)을 리스트 형태로 정의하는 문서입니다.

BinData/ 폴더: 문서에 삽입된 이미지 파일, 도형, OLE 객체 등 바이너리 데이터가 원본 형태로 저장됩니다.

Preview/ 폴더: 문서를 열지 않고도 내용을 미리 볼 수 있도록 썸네일 이미지(PrvImage.png)나 텍스트(PrvText.txt)가 저장됩니다.

2. Contents/ 폴더 (핵심 데이터)
문서의 실제 본문 내용, 서식, 메타데이터 등 문서 분석 시 가장 중요하게 다뤄지는 XML 파일들이 모여 있는 곳입니다.

content.hpf: 패키지 전체의 명세서 역할을 합니다. 문서의 메타데이터(작성자, 제목 등), 파일 구성 요소(Manifest 아이템), 파일을 읽는 순서(Spine)를 정의합니다.

header.xml: 문서 전반에 적용되는 서식 정보를 담고 있습니다. 글꼴, 문단 스타일, 글자 모양, 표의 테두리 및 배경, 개요 번호 매기기 등이 이 파일에 선언됩니다. 본문(section)에서는 이곳에 부여된 ID 값을 참조하여 스타일을 적용합니다.

section0.xml, section1.xml, ...: 실제 본문 텍스트와 객체가 담긴 핵심 파일입니다. 구역(Section)을 나눌 때마다 파일 숫자가 증가합니다.

텍스트, 표, 그림의 위치 정보 등이 모두 XML 태그 형태로 기록되어 있습니다. (예: 문단은 <hp:p>, 실제 텍스트는 <hp:t> 태그 내부에 위치)

settings.xml: 문서의 보기 배율, 호환성 설정, 기본 언어 등 편집기가 문서를 열 때 필요한 환경 설정 정보를 담고 있습니다.
